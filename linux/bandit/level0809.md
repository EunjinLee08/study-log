## 💡 풀이

### 핵심 개념

- `uniq`는 **인접한 중복 줄**을 제거하는 명령어다. 반드시 **정렬된 상태**에서만 제대로 동작한다.
  - 중복 여부를 판단할 때 바로 인접한 줄끼리만 비교하기 때문에, 정렬 없이 쓰면 떨어져 있는 중복을 잡지 못한다.
- `sort`로 먼저 정렬한 뒤 `uniq -u`를 적용하는 것이 기본 패턴이다.

| 명령어 | 의미 |
|--------|------|
| `sort` | 줄을 알파벳/사전 순으로 정렬 |
| `uniq` | 인접한 중복 줄 제거 |
| `uniq -u` | 중복이 **전혀 없는** 줄만 출력 |
| `uniq -d` | 중복된 줄만 출력 |
| `uniq -c` | 각 줄이 몇 번 등장하는지 앞에 숫자로 표시 |

### 명령어

```bash
# sort로 정렬 후 uniq -u로 유일한 줄만 출력
sort data.txt | uniq -u
```

### 실행 결과

```bash
bandit8@bandit:~$ sort data.txt | uniq -u
```

## 📝 메모

- `uniq`는 정렬 없이 쓰면 틀린 결과가 나오는 대표적인 함정을 갖는 명령어다. `sort | uniq`를 항상 세트로 외워두는 게 좋다.
- `sort`의 주요 옵션:
  - `-r` : 내림차순 정렬
  - `-n` : 숫자로 정렬 (기본은 문자열 정렬)
  - `-u` : 정렬하면서 중복 제거 (`sort -u`만으로 `sort | uniq`와 동일한 결과를 낼 수 있음)
