# Bandit Level 1 → Level 2

## 🎯 목표

홈 디렉토리에 있는 `-`라는 이름의 파일에서 다음 레벨의 비밀번호를 찾는다.

## 🔑 접속 정보

| 항목 | 값 |
|------|----|
| Host | `bandit.labs.overthewire.org` |
| Port | `2220` |
| Username | `bandit1` |
| Password | 이전 스테이지에서 확인 |

## 💡 풀이

### 핵심 개념

- 쉘에서 `-`는 파일 이름이 아니라 **stdin(표준 입력)** 을 의미하는 특수 기호로 해석된다.
  - 그래서 `cat -`를 입력하면 파일을 읽지 않고 키보드 입력 대기 상태가 된다.
- 이런 특수문자 이름의 파일을 읽으려면 두 가지 방법이 있다.
  1. **리디렉션(`<`)** 으로 파일을 stdin으로 넘기기: `cat < -`
  2. **경로를 명시**해서 파일 이름임을 분명히 하기: `cat ./-`

### 명령어

```bash
# 방법 1: 리디렉션 사용
cat < -

# 방법 2: 상대경로로 명시 (더 직관적)
cat ./-
```

### 실행 결과

```bash
# cat - 입력 시 → 아무 일도 일어나지 않음 (stdin 대기 상태)
bandit1@bandit:~$ cat -
▌  ← 커서만 깜빡이며 입력 대기

# cat < - 입력 시 → 비밀번호 출력
bandit1@bandit:~$ cat < -
rRGizSaX8Mk1RTb1CNQoXTcYZWU6lgzi
```

## 📝 메모

- `-`가 파일 이름으로 쓰이는 경우는 흔치 않지만, 쉘이 특수 의미로 먼저 해석하는 문자들(`-`, `~`, `*`, `?` 등)은 항상 주의가 필요하다.
